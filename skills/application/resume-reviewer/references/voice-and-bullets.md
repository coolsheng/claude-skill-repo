# Voice & achievement bullets

How to make every bullet earn its place and the whole resume speak with one voice at the target seniority.

## The achievement-bullet formula

Strong bullets show **outcome**, not duty. Useful templates:

- **XYZ (Google formula):** "Accomplished [X] as measured by [Y] by doing [Z]." → *"Cut API p99 latency 40% by introducing a Redis read-through cache."*
- **Action → Impact → (How):** strong verb → quantified result → method/scope.

Each bullet should answer: *What changed because you were there?*

### Before → after

- ❌ "Responsible for managing the deployment pipeline."
  ✅ "Cut deploy time from 25 to 6 min by parallelizing CI stages, unblocking ~30 daily releases."
- ❌ "Worked on improving customer onboarding."
  ✅ "Redesigned onboarding flow, lifting activation 18% (12k→14.2k MAU) over two quarters."

## Quantify — but never fabricate

- Prefer numbers: %, $, time saved, scale (users/QPS/rows), team size, count shipped.
- If no metric exists, quantify scope or use a concrete qualitative outcome ("adopted by all 4 product teams"). **If a number is missing, ask the user — do not invent one.**

## No editorializing / AI-tells

State facts; don't pitch them. Resume copy that *sells* a fact reads as AI-generated and unprofessional — recruiters notice. Watch especially **credential, visa, and contact lines**:

- ❌ "Eligible for E-3 visa (Australian national — no lottery, low-cost/fast for employers)"
  ✅ "Eligible for E-3 visa (streamlined US work authorization for Australian nationals)"
- Avoid editorializing parentheticals, slash-lists ("low-cost/fast"), and benefit-selling asides. One neutral descriptor (e.g. "streamlined") carries the signal; the reader draws the conclusion.
- This applies beyond visa lines — anywhere the resume explains *why something is good* rather than stating *what it is* is a candidate for cutting.

### Punctuation tells: no em-dashes in resume output

Em-dashes (—) read as an AI tell; resume output must not contain them. Replace case-by-case with a colon, parentheses, a comma, or a `·` separator (in header/title lines). **En-dashes in date ranges ("Jun 2019 – Present") are standard typography — keep them.** This rule applies to the resume artifact only, not to reviews or chat.

## Verbs

- Lead every bullet with a strong past-tense action verb (Led, Built, Shipped, Reduced, Negotiated, Architected, Automated).
- Avoid weak/filler openers: *responsible for, worked on, helped with, assisted, duties included, tasked with, involved in*.
- Don't repeat the same verb across many bullets — vary while staying accurate.

## Tense rule (consistency)

- Current role: present tense. Past roles: past tense. Apply this rule uniformly — mixed tense within one role is a red flag.

## No first person

- Drop pronouns entirely ("Led…", not "I led…"). Implied first person throughout.
- Where prose makes a pronoun unavoidable (e.g. a summary sentence), use third person ("his own product"), never "I/my".

## Voice that matches seniority

The resume should read like the level it targets — consistent across all entries:

- **Junior:** emphasize delivery, concrete skills, learning velocity, ownership of well-defined tasks. Don't inflate scope into leadership you didn't have.
- **Mid:** emphasize end-to-end ownership of features/projects, cross-functional work, measurable impact.
- **Senior/Lead:** emphasize ambiguity, technical direction, multiplier effects (mentoring, standards), and business outcomes — not just code. Bullets should operate at system/org scope, not task scope.

Flag **mismatch in both directions**: a senior candidate whose bullets read like task tickets (undershoot), or a junior candidate claiming org-wide leadership (overshoot). One coherent register across the whole document.

## Voice-consistency checklist

- [ ] Same tense rule applied everywhere.
- [ ] Every bullet verb-led, no filler openers.
- [ ] No first-person pronouns.
- [ ] Parallel grammatical structure within each entry's bullets.
- [ ] Consistent terminology for the same tech/role/skill.
- [ ] Register (scope of claims) consistent and matched to target seniority.
- [ ] Quantification present wherever a real number exists.
- [ ] No editorializing/AI-tells — facts stated, not pitched (esp. credential/visa lines).
- [ ] No em-dashes in the resume — colon/parens/comma/`·` instead (date-range en-dashes OK).
